STM32 安全机制概览

发表于 2019-07-26 分类于 安全

安全目标

需要保护的资产:
| Type of attack | Targeted assets | Risks |
|:—|:—|:—|
| Data access | Sensor data(healthcare data, log of postions…), User data(ID, PIN, password, accounts…), Transactions logs, Cyptographic keys | Unauthorized sale of personal data, Usurpation(篡夺), Spying, Blackmail |
| Control of device | Device correct functionality, Device/User identity | Denial of service, Attacks on service providers, Fraudulent(欺骗) access to service |
| Reverse engineering | Device hardware architecture/design, Software patent/architecture, Tech patents | Device counterfeit, Software counterfeit, Software modification, Access to secure areas |

安全系统组成

为了对抗多种攻击，数种安全机制必须在系统内实现，全面的安全需要从硬件到应用程序通篇考虑。一方面，如果设备没有对外部入侵（如调试接口）的保护，敏感数据的应用就无法被可信的实现于硬件平台上。另一方面，健壮的设备保护机制在潜在的应用漏洞（如允许访问设备全部地址空间）面前是无效的。

完整的设备保护机制包括数种存储保护机制，使其免于外部接口，控制启动模式和设备环境监视。典型的安全应用包括：

  • 安全启动，保证启动时应用完整可靠未篡改
  • 支持安全的通信协议（如 TLS）
  • 安全的固件更新能力，以安全的方式下载新的固件版本
  • 安全密钥存储（Secure Key Storage，SKS）能力

攻击类型

对微控制器的攻击分类如下：

  • 软件攻击：利用软件弱点（如协议弱点）
  • 硬件非侵入攻击：专注于利用微控制器接口和环境信息
  • 硬件侵入攻击：破坏性的接触硅片的攻击方式

软件攻击
